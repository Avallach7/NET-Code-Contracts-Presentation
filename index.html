// The .NET Foundation licenses this file to you under the MIT license. namespace System.Diagnostics.Contracts { public static class Contract { event EventHandler<ContractFailedEventArgs> ContractFailed { add { } remove { } } void Assert(bool condition, string userMessage) { } void Assume(bool condition, string userMessage) { } void Ensures(bool condition, string userMessage) { } void EnsuresOnThrow<TException>(bool condition, string userMessage) where TException : Exception { } bool Exists<T>(IEnumerable<T> collection, Predicate<T> predicate) { ... } bool ForAll<T>(IEnumerable<T> collection, Predicate<T> predicate) { ... } void Invariant(bool condition, string userMessage) { } T OldValue<T>(T value) { ... } void Requires(bool condition, string userMessage) { } void Requires<TException>(bool condition, string userMessage) where TException : Exception { } T Result<T>() { ... } T ValueAtReturn<T>(out T value) { ... } } public class PureAttribute : Attribute { } }
  • Designed to do nothing when runtime checking is off (no performance overhead!)
  • Assertion methods are compiled conditionally: [Conditional("DEBUG"), Conditional("CONTRACTS_FULL")] void Ensures(bool condition, string userMessage) {...}
  • Helper methods should only be used as their inline parameters: // returns, so cannot be [Conditional] bool ForAll<T>(IEnumerable<T> collection, Predicate<T> predicate) {...} Contract.Requires(Contract.ForAll<int>(inputs, i => i > 0)); // both will be either rewritten or omitted
  • "Hacking" Contract methods to run without rewriter results in unspecified behaviour: bool inputsValid = Contract.ForAll<int>(inputs, i => i == i); // ERROR (may e.g. always return true) #define DEBUG #define CONTRACTS_FULL Contract.Requires(true); // ERROR (may e.g. do nothing, throw or display GUI error window)
  • Integrated with IDE (VS 2010-2015, produces ReSharper-like warnings)
  • Suggests contracts where they are not declared but can be deduced (implicit contracts): CodeContracts: Suggested requires: Contract.Requires(inputArray != null);
  • Conditional compilation symbols are defined: #define DEBUG #define CONTRACTS_FULL
  • ContractsRuntime class is injected into assembly
  • Calls to Contract.* are replaced by calls to ContractsRuntime.*
  • ContractsRuntime methods do actual check and react on failure
    • call System.Runtime.CompilerServices.ContractHelper.RaiseContractFailedEvent12
    • which calls registered handlers and native runtime-specific method (e.g. displaying error window)34
    • if not handled, throws System.Diagnostics.Contracts.ContractsRuntime.ContractException
